9. Character arrays and pointers - part 2

記憶體配置解析與唯讀記憶體區段

🧠

記憶體與Stack解析

當我們將一個字元陣列作為參數傳遞給函式時，實際發生了什麼事？理解記憶體堆疊 (Stack) 的運作方式至關重要。

💡 重點筆記
  • 當呼叫 print(C) 時，陣列 C 會自動轉換成指向其首元素 (C[0]) 的指標。
  • 在 print() 函式的堆疊框架 (stack frame) 中，會建立一個新的指標變數 C，這個指標的值就是 main 函式中陣列的起始位址。
  • 函式結束後，print() 的堆疊框架（包括其內部的指標變數 C）會被釋放，但 main 函式中原始的陣列內容仍然存在。
🔒

唯讀記憶體 (Read-Only Data Segment)

在處理字串時，char C[] = "Hello"; 和 char *C = "Hello"; 這兩種寫法有著本質上的區別，這關乎到字串常數的儲存位置。

範例 9.2.1 - 兩種字串宣告方式
#include <stdio.h>

int main() {
    // 寫法一：字串存放在 Stack 區域的陣列中，內容可修改
    char C1[] = "Hello";
    C1[0] = 'A';
    printf("C1: %s\n", C1); // 輸出 "Aello"

    // 寫法二：字串 "Hello" 是常量，存放在唯讀記憶體區
    char *C2 = "Hello";
    // C2[0] = 'A'; // ❌ 錯誤！這會試圖修改唯讀記憶體，導致執行期錯誤 (Segmentation Fault)
    printf("C2: %s\n", C2);

    return 0;
}
💡 重點總結
  • 陣列宣告 (char C[])：會在 Stack 上建立一個新的字元陣列，並將字串常數的內容**複製**進去。這個陣列的內容是**可以修改**的。
  • 指標指向字串常量 (char *C = "...")：會讓指標 C 直接指向儲存在**唯讀記憶體區段 (.rodata)** 的字串常數 "..."。任何試圖透過這個指標修改字串內容的行為都是未定義的，通常會導致程式崩潰 (Segmentation Fault)。

為了安全起見，當使用指標指向一個字串常數時，最好將指標宣告為 const char *，這樣編譯器就會在編譯時期檢查並防止你意外地修改它。

安全寫法
const char *C = "Hello";
// C[0] = 'A'; // ✅ 編譯器會直接報錯，提前防止執行期錯誤
